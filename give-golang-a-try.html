Give golang a try

Posted on Thu 28 March 2019 in programming

“Go is such a refreshing language to program in, there is very little clutter just the stuff you need to get the job done” – Matt West

Background

I've bean using java and python for years. With spring boot, it is easy to start an application. However, I still don't like the database access part of java.

Now JPA and mybatis are popular solutions for database access. When I need to write some queries with dynamic conditions, the code is still ugly.

In my recent projects, because we have very limited resource (Only 1 developer for both API and frontend), I begin to use Django and Django rest framework to replace spring boot.

Actually, most of our APIs are just simple CRUD. With Django rest framework and django-filter, I only need a few lines of codes to get the API done.

One of our applications is to collect data from some machine tools, do some calculation and show some reports. It suppports uploading data through both some HTTP endpoints and MQTT.

For MQTT, integrating MQTT with Django is difficult. So I use hbmqtt and another simple python file to receive the data from mosquitto. Because we only need to collect data from a few machine tools (< 10), this file is enough.

However, we just got a big project which has about 100 machine tools and more than 1000 data points (Some data points' sampling rate is 100Hz). Then I begin to worry about the performance of this python file. I decide to give golang a try because it is hot recently.

Things I don't like

  1. Because I only use C during my college, the pointer is a little confusing for me
  2. For a modern language, golang is a little dull. No list comprehension, no pattern match, if err != nil is everywhere

Things I like

  1. Simple. The core of the language is simple enough just like C.
  2. Flexible. Don't have the limitation like "one public class in one file", so with only one or two files, you can get a fully workable application.
  3. Static typed. Feel more confident to change my code than python.
  4. Can build to a single executable file. It is so easy to deploy.
  5. Don't need a heavy IDE to write code. I can use my favorite emacs now.

After some google works, I'm able to make my golang version MQTT receiver works. Just as the saying goes, it just works.
